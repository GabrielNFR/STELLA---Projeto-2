## Sobre o Projeto

**STELLA** é uma aplicação web desenvolvida para apoiar pacientes em tratamento de Fertilização In Vitro (FIV) da Clínica AMARE. A plataforma centraliza informações, medicações, consultas e documentos, devolvendo à paciente a sensação de controle e segurança durante todo o processo.

<details>
<summary><strong>Problema que Resolvemos</strong></summary>

### Desafios Enfrentados pelas Pacientes

Pacientes de FIV enfrentam:
- Protocolos complexos entregues em PDFs estáticos
- Comunicação fragmentada via WhatsApp, ligações e papel
- Ansiedade pela falta de visibilidade da jornada
- Dificuldade em gerenciar múltiplas medicações e sintomas
- Sobrecarga cognitiva de memorizar horários, dosagens e etapas

### Nossa Solução

Uma plataforma web mobile-first que:
- Visualiza a jornada completa em linha do tempo
- Gerencia medicações com alertas inteligentes
- Registra sintomas e emoções de forma integrada
- Oferece suporte emocional e informações confiáveis
- Permite participação ativa do parceiro (modo co-piloto)

</details>

## Épicos do Produto

<details>
<summary><strong>Épico 1: Acompanhar Minha Jornada de Tratamento</strong></summary>

**Objetivo:** Transformar a complexidade do tratamento em uma jornada clara e visível.

### Funcionalidades:
- **Linha do tempo interativa** | Visualização de todas as etapas do tratamento
- **Agenda de consultas e exames** | Notificações automáticas e lembretes
- **Modo co-piloto** | Permite que o parceiro acompanhe e apoie ativamente

### User Stories:
- H1.1: Visualizar linha do tempo do tratamento
- H1.2: Adicionar consulta ou exame na agenda
- H1.3: Ativar modo co-piloto para parceiro

</details>

<details>
<summary><strong>Épico 2: Gerenciar Minhas Medicações e Saúde</strong></summary>

**Objetivo:** Integrar medicações, sintomas e estados emocionais em um sistema inteligente.

### Funcionalidades:
- **Registro integrado** | Medicação + sintomas + emoções em uma única tela
- **Diário de emoções** | Acompanhamento do bem-estar ao longo do tempo
- **Alertas inteligentes** | Notificações no momento certo de tomar medicação

### User Stories:
- H2.1: Registro integrado de medicação e sintomas
- H2.2: Diário de emoções
- H2.3: Alertas de medicação

</details>

<details>
<summary><strong>Épico 3: Receber Suporte e Informações</strong></summary>

**Objetivo:** Garantir que nenhuma paciente se sinta sozinha ou desamparada.

### Funcionalidades:
- **Suporte emocional rápido** | Acesso 24h a recursos de apoio
- **Biblioteca educativa** | Materiais confiáveis organizados por categoria

### User Stories:
- H3.1: Acesso rápido a suporte emocional
- H3.2: Visualização de materiais educativos

</details>

## Metodologia

<details>
<summary><strong>Desenvolvimento Ágil | Scrum</strong></summary>

### Estrutura
- 2 Sprints de 1 semana cada
- User Stories com critérios de aceitação
- Cenários BDD (Behavior-Driven Development)
- Correlação Design e Desenvolvimento em todas as entregas

### Sprint 1 (Semana 1): MVP Essencial
**Objetivo:** Criar base funcional do sistema

**User Stories planejadas:**
- H1.1: Visualizar linha do tempo
- H1.2: Adicionar consulta na agenda
- H2.1: Registrar sintoma físico

**Entregas esperadas:**
- Design: Wireframes, paleta de cores, componentes principais
- Desenvolvimento: Estrutura do banco, autenticação, APIs básicas, componentes frontend

### Sprint 2 (Semana 2): Funcionalidades Críticas
**Objetivo:** Completar MVP e funcionalidades prioritárias

**User Stories planejadas:**
- H2.2: Registrar emoções no diário
- H2.3: Alertas de medicação
- H3.1: Suporte emocional
- H3.2: Materiais educativos

**Entregas esperadas:**
- Design: Protótipos navegáveis, testes de usabilidade
- Desenvolvimento: Sistema de notificações, busca, integração completa, testes

## Como Executar

<details>
<summary><strong>Instalação e Execução Local</strong></summary>

### Pré-requisitos
- Python 3.10+
- pip
- Git

### Passo a Passo

```bash
# Clone o repositório
git clone https://github.com/seu-usuario/STELLA-Projeto-2.git
cd STELLA-Projeto-2

# Crie um ambiente virtual
python -m venv venv
source venv/bin/activate  # Linux/Mac
# ou
venv\Scripts\activate  # Windows

# Instale as dependências
pip install -r requirements.txt

# Execute as migrações do banco
python manage.py migrate

# Inicie o servidor
python manage.py runserver

# Acesse no navegador
```
